報警資料檢查與重標註 GUI：操作說明

本工具適合在模型驗證或 shadow mode 期間使用。它不會連 RTSP、不會寄 Email，也不會載入 best.pt；它只處理已保存的告警影像與 JSON。

建議第一次使用：先複製資料集到備份位置，先「保留在原地」測試幾筆，確認輸出正確後再使用「移動」或「刪除」。

1. 安裝與啟動

cd industrial-smoke-monitor
python -m pip install -r alert_reviewer/requirements.txt
python alert_reviewer/gui/alert_reviewer_gui.py \
  --input data/runtime/alerts \
  --dataset data/processed/field_yolo \
  --split train

PowerShell 的換行符號是反引號，也可以把指令寫成一行。GUI 需要 Python 3.11.15 與 Tkinter；不需要安裝 Ultralytics、PyTorch 或 GPU。

2. 畫面上的基本操作

左側清單

列出每個告警 JPG／JSON。unreviewed 表示尚未人工確認；點選後中央會顯示影像。

中央影像

影像已包含模型輸出的 ROI、box、mask、conf、area、cls。依序點擊煙霧輪廓，再按「完成目前 polygon」可建立人工標註。

右側資訊

顯示 timestamp、整體 conf／area、instance 數、ROI 與每個模型 instance 的 box／polygon；它是判讀模型的證據。

  1. 先選擇人工分類：有煙選 smoke；雲、蒸汽、光影或其他誤報選 no_smoke。
  2. 若是 smoke，檢查原模型 polygon；錯誤時按「清除全部線條」後在影像上重新點出輪廓。
  3. 必要時修改上方「訓練資料集」路徑與 split。正常新增現場樣本使用 train；不要把同一段影片的相鄰影格分到 val/test。
  4. 選擇完成後處理，再按「套用目前告警」。

3. 三種完成後處理

選項會發生什麼適合何時使用
保留在原地更新 JSON 的 review、分類與 polygon，並在告警旁寫同名 YOLO .txt；原 JPG 不移動。先做品質檢查、還不想加入訓練資料時。
移動到訓練資料集寫入 images/{split}、labels/{split}，更新 dataset.yaml／manifest.csv，成功後刪除原告警 JPG、JSON、txt。人工已確認且希望下一輪訓練使用時。
刪除刪除該告警 JPG、JSON、txt，不會加入訓練集，也不會刪除其他 log。確定是無用資料或公司政策要求刪除時。
刪除與移動不可由 GUI 復原。若日後可能需要稽核，先備份 data/runtime/alerts/。即使選擇 move，程式只會移除該筆 alert artifact，不會清除母專案的 detections.jsonl。

4. 如何判斷一筆告警

觀察項目新手可問的問題
影像內容煙霧是否真的從煙囪出口出現？雲、蒸汽、雨、逆光與熱氣是否被誤認？
ROI煙囪和煙霧是否在 ROI 內？ROI 是否太窄而截斷煙霧，或太大而包含天空？
mask橘／紅色輪廓是否覆蓋煙霧，而不是只框住煙囪、建築或背景？
conf 與 area高 conf 不等於一定正確；area 是 mask 佔 ROI 的比例，需結合影像與連續影格一起看。
跨影格同一段煙霧是否有多個告警？可用 timestamp 判斷是否只是每秒抽樣的同一事件。

5. 匯入訓練與資料品質

python dataset_builder/scripts/check_dataset.py \
  --dataset data/processed/field_yolo

python scripts/train.py \
  --data data/processed/field_yolo/dataset.yaml \
  --model yolo11n-seg.pt \
  --name smoke_field_v002

如果 GUI 中標記 no_smoke，輸出的是空的 label 檔，這是 YOLO 表示負樣本的正確方式。若標記 smoke 卻沒有 polygon，工具會提醒；除非你有明確理由，否則不要把沒有輪廓的影像當作 smoke 訓練。

  • 同一段影片的連續影格很相似，不能只因樣本數增加就認為泛化能力增加；按攝影機／日期／影片切分。
  • 保留一批從未加入訓練的現場影片作為固定驗收集，否則每輪訓練都可能只是在記住已看過的影像。
  • 記錄模型版本、資料集版本、門檻與現場事件結果；不要覆蓋唯一的 best.pt。

6. 常見問題

清單是空的

確認輸入資料夾是 data/runtime/alerts，並且每筆資料有同名 JPG 與 JSON。新的 monitor 只有在 SAVE_ALERT_SNAPSHOTS=true 且偵測為 smoke 時才會產生。

看不到原模型 polygon

舊版本只保存簡單候選 JPG 時，JSON 可能沒有 instances[].polygon。你仍可以在中央影像重新點 polygon；新版本會完整保存逐 instance polygon。

能不能直接把 alert 影像丟到資料集？

可以用 GUI 的 move，但不建議人工未確認就批量搬入。誤報會成為錯誤標籤，下一輪模型可能學會錯誤背景。

GUI 是否會寄信？

不會。Email 只由母專案 monitor 負責；reviewer 不讀 SMTP 設定，也不載入模型。
