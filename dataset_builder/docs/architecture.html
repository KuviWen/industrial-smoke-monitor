Industrial Smoke Dataset Builder

這個子專案負責把「影片」製作成母專案可以直接訓練 YOLO11 instance segmentation 的資料。它把抽幀、人工確認、可選的模型預標註、資料檢查與資料集合併分開，方便在隔離外網的 Windows 電腦上重複使用。

一、整體資料流

1. 影片
mp4 / avi / mov / 3gp
攝影機錄影或匯出檔
→
2. 取樣
OpenCV 依秒數抽幀
暫存到 staging
→
3. 標註與確認
Tkinter 點選 polygon
可用 YOLO11-seg 預標註
→
4. YOLO 資料集
images / labels / YAML
直接給母專案訓練

重要：抽出的影格只是候選資料，不會自動被當成「無煙」。只有在 GUI 中確認後按下儲存，才會寫入最終資料集；沒有 polygon 的人工確認影格才會產生空白 label。

左右翻轉資料增強

在 GUI 勾選「是否同時生成左右翻轉的副本」後，每次儲存一張已人工確認的影格，writer 會把原始影像與 polygon 一起複製成另一個訓練樣本。這不是只翻轉圖片：label 中的每個 polygon 也會同步鏡射。

原始 image + polygons
        │
        ├── 原始 image.jpg + image.txt
        │
        └── cv2.flip(image, 1)
            x' = width - 1 - x
            → image_flip.jpg + image_flip.txt

影像座標的有效 x 範圍是 0 ... width-1，所以水平翻轉使用 x' = width - 1 - x，y 座標維持不變。翻轉副本與原始樣本會放在相同的 split，並各自留下 manifest.csv 紀錄；副本的狀態為 reviewed_augmented_horizontal_flip。

資料評估原則：這個選項只建議用於 train。GUI 對 val / test 會阻止儲存，避免驗證與測試資料被資料增強，造成評估結果偏樂觀。

三、最後輸出與母專案的對接

data/processed/video_yolo/
├── dataset.yaml
├── manifest.csv
├── images/{train,val,test}/*.jpg
└── labels/{train,val,test}/*.txt

dataset.yaml 會用輸出資料夾的絕對路徑，避免 Windows 從不同工作目錄啟動時找不到影像。若整個資料夾搬到另一台電腦，執行 create_dataset_yaml.py 重新生成即可。

若啟用翻轉增強，同一筆人工標註會在相同 split 多出一組以 _flip 結尾的 image/label；資料夾的配對規則不變，母專案不需要額外設定。

檔案用途母專案使用方式
images/train 等實際影像Ultralytics 讀取
labels/train 等每個影像同名的 segmentation label一個 polygon 一行；類別 0 是 smoke
dataset.yaml資料根目錄、split 與類別名稱python scripts/train.py --data data/processed/video_yolo/dataset.yaml
manifest.csv影片來源、frame index、時間、label 數量稽核與日後追查，不是 YOLO 必需檔

四、YOLO11-seg label 長什麼樣子？

0 0.215000 0.180000 0.280000 0.165000 0.340000 0.240000
0 0.620000 0.330000 0.700000 0.290000 0.760000 0.390000

第一個數字是類別 id；後面是依序排列的 polygon 頂點，x、y 都已除以影像寬、高，範圍通常為 0 到 1。若一張影像經人工確認沒有煙，label 檔保留為空白檔案。

五、為什麼 split 要以影片/場次為單位？

同一支影片相鄰影格非常相似。若把同一支影片隨機切到 train 與 val，驗證分數可能看似很高，但其實模型只是看過幾乎相同的畫面，這叫資料洩漏。GUI 的 split 要在處理整支影片前決定，建議一支影片只進一個 split，例如：

來源用途
不同日期、攝影機或場次分散到 train / val / test，讓測試更接近真正換場景
同一支影片的全部影格只能選一個 split
每個 split 沒有樣本先補資料；不要拿同一段影片複製到另一個 split

六、套件與離線設計

GUI 使用 Python 內建 Tkinter，不強制加入 Streamlit、Qt 或其他 GUI 套件。影像讀寫由 OpenCV 處理，預覽由 Pillow 顯示，YOLO11 自動預標註則在使用者提供本機 .pt 權重後才載入。這樣在隔離外網環境中，手動標註仍可運作，而且不會因為嘗試下載權重而卡住。

和母專案的關係：這是母專案的資料製作工具，不會修改 RTSP、告警或 Email 程式。產生的 data/processed/video_yolo/dataset.yaml 可直接作為母專案訓練輸入。
