Industrial Smoke Dataset Builder 操作說明書

目標：把一支或多支影片中的煙囪畫面，整理成母專案可直接訓練的 YOLO11 segmentation 資料。以下命令以 Windows Anaconda Prompt、母專案根目錄為例。

0. 先理解兩種資料夾

資料夾用途能不能直接拿去訓練
dataset_builder/data/stagingextract_frames.py 產生的候選影格不能；尚未人工確認
data/processed/video_yoloGUI 儲存的最終 YOLO 資料集可以；先跑檢查

不要跳過人工確認：如果候選影格其實有煙卻被寫成空白 label，模型會學到「有煙也算背景」，這通常比少一張資料更傷害效果。

1. 確認 Python 與環境

在母專案根目錄（可看到 scripts/ 與 dataset_builder/）開啟 Anaconda Prompt：

conda activate YOLO11
python --version
python -c "import sys; print(sys.executable)"
python -c "import cv2, numpy, yaml, PIL, torch, torchvision, ultralytics; print('imports OK')"
python -m tkinter

Python 版本應為 3.11.15。最後一行會開一個小視窗；關閉它即完成 Tkinter 測試。若環境名稱不同，替換 conda activate 後面的名稱即可。

子專案的依賴版本與需求一致：ultralytics>=8.3.0,<9.0、opencv-python>=4.9.0,!=4.13.0.90、numpy==1.26、PyYAML>=6.0、python-dotenv>=1.0、Pillow>=10.0、torch==2.6.0+cpu、torchvision==0.21.0+cpu。若母環境已安裝，不必重裝。

2. 準備影片

將影片放在不會被 Git 提交的位置，例如：

data/raw/videos/chimney_2026-07-18_01.mp4

檔案可以是攝影機錄影匯出的 mp4/avi/mov/mkv/ts/3gp。此子專案的輸入是「影片檔」；RTSP 串流若要製作資料，請先用母專案或錄影工具保存成影片檔，以便之後重現來源。.3gp 能否讀取取決於 OpenCV/FFmpeg 後端是否包含 3GPP codec。

3. （可選）先抽幀查看候選資料

python dataset_builder/scripts/extract_frames.py \
  --video data/raw/videos/chimney_2026-07-18_01.mp4 \
  --output dataset_builder/data/staging \
  --every-seconds 1

Windows CMD 可把每行結尾的 \ 改成一行完整命令；PowerShell 使用反引號 `。--every-seconds 1 表示每秒取一張，不代表每秒的影格都一定要存進最終資料集。

候選資料會在 dataset_builder/data/staging/<影片名>/frames/。這個步驟不是必要步驟；GUI 也可以直接讀原始影片並跳到指定影格。

4. 啟動 GUI 並決定 split

python dataset_builder/gui/annotator_gui.py \
  --video data/raw/videos/chimney_2026-07-18_01.mp4 \
  --output data/processed/video_yolo \
  --split train

如果要使用已訓練的本機 YOLO11-seg 權重做預標註：

python dataset_builder/gui/annotator_gui.py \
  --video data/raw/videos/chimney_2026-07-18_01.mp4 \
  --output data/processed/video_yolo \
  --weights models/best.pt \
  --split train

權重必須是 segmentation 模型，例如母專案訓練出的 best.pt。離線環境下不要只寫 yolo11n-seg.pt 期待它自動下載；請先把權重檔放到本機。

GUI 操作

  1. 影片整個場次開始前先選好 train、val 或 test。同一支影片不要中途換 split。
  2. 在畫面上依煙霧外輪廓逐點點擊；至少 3 點後按「完成多邊形」。一張畫面可畫多個煙霧區域。
  3. 有本機權重時按「載入/自動預標註」；它只提供起點，仍要人工檢查、清除或重畫。
  4. 確定有煙，保存 polygon；確定沒有煙，保持 0 個 polygon 後按「儲存此影格」，GUI 會建立空白 .txt。
  5. 用「上一影格」「下一影格」或輸入影格索引跳轉。按儲存後再繼續下一張。

左右翻轉資料增強

如果這張影格已經人工確認完成，而且要把它加入訓練資料的左右翻轉版本，勾選「是否同時生成左右翻轉的副本」再按「儲存此影格」。一次儲存會產生原始檔與副本：

images/train/chimney_01_f00000100.jpg
labels/train/chimney_01_f00000100.txt
images/train/chimney_01_f00000100_flip.jpg
labels/train/chimney_01_f00000100_flip.txt

工具會使用同一張影像的寬度，把所有 polygon 的 x 座標轉成 width - 1 - x，再寫出翻轉 label；你不需要重新畫遮罩。原始與翻轉影像都會出現在 manifest.csv。空白 polygon 的負樣本也會產生空白的翻轉 label。

只對 train 使用：如果目前 split 是 val 或 test，GUI 會顯示警告並停止這次儲存；請切換回 train，或取消勾選後再儲存。

5. 檢查資料集

python dataset_builder/scripts/check_dataset.py \
  --dataset data/processed/video_yolo

檢查項目包括：三個 split 的目錄、每張影像是否有同名 label、label 的類別 id、polygon 是否至少三個頂點、座標是否在 0 到 1，以及資料集是否至少有一張影像。成功時會重新寫出 dataset.yaml。

若資料夾搬到別台電腦，先重新執行：

python dataset_builder/scripts/create_dataset_yaml.py \
  --dataset data/processed/video_yolo

6. 直接交給母專案訓練

python scripts/train.py \
  --data data/processed/video_yolo/dataset.yaml \
  --model yolo11n-seg.pt \
  --epochs 100 \
  --imgsz 640 \
  --batch 8 \
  --device cpu \
  --name smoke_video_yolo11

現有的 CPU PyTorch 環境可以跑流程，但訓練可能很慢。若在有 NVIDIA GPU 的訓練電腦上執行，另建與驅動/CUDA 相容的 GPU 環境，並把 --device cpu 改成 --device 0；不要為了現場 CPU 環境任意替換 torch wheel。

scripts/train.py 會把相對的輸出路徑固定解析到母專案根目錄，因此輸出會維持在 runs/train/smoke_video_yolo11/，不會因為從 runs/segment 等資料夾啟動而變成巢狀路徑。

若要先以測試集驗證：

python scripts/validate.py \
  --weights runs/train/smoke_video_yolo11/weights/best.pt \
  --data data/processed/video_yolo/dataset.yaml \
  --split test \
  --device cpu

7. 將 IJmond 與影片資料集合併（可選）

python dataset_builder/scripts/merge_yolo.py \
  --source data/processed/ijmond_yolo data/processed/video_yolo \
  --output data/processed/smoke_combined_yolo

合併工具會替檔名加上來源資料集名稱並處理重名，不會覆蓋來源資料。合併後先檢查：

python dataset_builder/scripts/check_dataset.py \
  --dataset data/processed/smoke_combined_yolo

python scripts/train.py \
  --data data/processed/smoke_combined_yolo/dataset.yaml \
  --model yolo11n-seg.pt \
  --device cpu \
  --name smoke_combined_yolo11

常見問題

現象處理
GUI 開不起來或顯示 cv2/PIL 缺少確認目前 Prompt 的 Python 是 conda activate YOLO11 後的 Python，再執行 import 測試；不要只在另一個環境安裝套件。
「自動預標註」嘗試下載檔案把本機完整路徑填入權重欄，例如 C:\\...\\models\\best.pt。本子專案的 GUI 不負責在隔離環境下載權重。
check_dataset 說缺少 label每張 image 都必須有同名 .txt；回到 GUI 儲存該影格，無煙影格也要存出空白檔。
訓練 loss 或驗證結果很差先檢查 polygon 是否真的圈住煙，負樣本是否人工確認，左右翻轉只放在 train，並確認同一支影片沒有同時出現在 train 與 val/test。
GitHub 變得很大不要提交影片、data/processed、staging、*.pt 與 runs/。Git 只保存程式、文件和測試。

GitHub 提交流程

git status
git add dataset_builder README.md
git commit -m "Add video dataset builder for YOLO11 smoke segmentation"
git push

建議在提交前執行 python -m compileall dataset_builder/src dataset_builder/scripts dataset_builder/gui，並在已安裝完整依賴的環境執行測試。原始影片與模型權重透過另外的受控儲存管理。
