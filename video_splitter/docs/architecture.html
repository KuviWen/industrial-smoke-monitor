Industrial Video Splitter

本子專案把一個影片檔切成一個或多個獨立片段，方便後續交給 dataset_builder/ 抽幀、標註與製作 YOLO11 smoke segmentation 資料。

一、資料流

影片檔
mp4 / avi / mov / mkv / 3gp
→
OpenCV 讀取
取得 FPS、尺寸、影格數
→
Tkinter GUI
預覽、選擇時間範圍
→
VideoWriter
重新編碼為 MP4

限制：OpenCV 的 VideoWriter 只寫影像串流，這個工具不複製音訊。輸出的片段適合影像資料製作；需要保留音訊時應使用獨立的 FFmpeg 流程。

三、兩種切割模式

模式輸入結果適用情境
單段開始時間、結束時間一個 prefix_0001.mp4只保留某次冒煙事件或一段特定畫面
批次開始/結束範圍、每段長度、起點間隔多個連續片段把長時間錄影切成容易人工檢查的小段

批次模式中，起點間隔等於每段長度時不重疊；起點間隔小於每段長度時會產生重疊片段。最後不足一個完整長度的尾段也會輸出，只要仍至少包含一個影格。

四、核心模組責任

模組責任
read_video_info檢查影片是否可開啟，取得寬、高、FPS、影格數與總長度。
parse_timecode / format_timecode在秒數與 HH:MM:SS.mmm 之間轉換，避免 GUI 直接處理字串細節。
build_segments驗證起訖範圍並建立單段或批次的半開區間 [start, end)。
split_video依影格索引讀取並用 VideoWriter 輸出；支援進度回報、覆寫控制與取消事件。
video_splitter_gui.py只負責操作畫面與背景執行緒，不把 GUI 事件與影片切割演算法混在一起。

五、和資料製作子專案的關係

video_splitter GUI 輸出 .mp4
        ↓
dataset_builder/scripts/extract_frames.py
        ↓
dataset_builder/gui/annotator_gui.py
        ↓
data/processed/video_yolo/

切割器不會替影片標註煙霧，也不會修改母專案模型。它只負責把長影片整理成可管理的來源片段。
