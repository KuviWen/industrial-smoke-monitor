Industrial Video Splitter 操作說明書

本工具以 GUI 切割本機影片檔，適合先把長時間的煙囪錄影切成小片段，再交給母專案的 dataset_builder/ 進行抽幀與 YOLO11 標註。

先說清楚：這個版本用 OpenCV 重新編碼影像，輸出檔通常不含音訊。它的設計目標是影像資料製作，不是影音剪輯或音訊保存。

1. 確認環境

從母專案根目錄開啟 Anaconda Prompt：

cd C:\industrial-smoke-monitor
conda activate YOLO11
python --version
python -c "import cv2, numpy, PIL; print('video dependencies OK')"
python -m tkinter

Python 版本應為 3.11.15。Tkinter 是標準 Python 的 GUI 元件，本子專案不新增 Streamlit、Qt 或其他視窗套件。

若母環境尚未有本子專案需要的套件，可在可連網環境使用：

python -m pip install -r video_splitter/requirements.txt

隔離電腦請沿用既有離線安裝流程，不要在現場假設可以下載套件。

2. 啟動 GUI

python video_splitter/gui/video_splitter_gui.py `
  --output video_splitter/data/output

也可以直接帶入影片：

python video_splitter/gui/video_splitter_gui.py `
  --video data/raw/videos/chimney_2026-07-18_01.mp4 `
  --output video_splitter/data/output

如果使用 Windows CMD，請把 PowerShell 的反引號 ` 改成單行命令，或移除換行。

3. 開啟與預覽影片

  1. 按「選擇影片」挑選 .mp4、.avi、.mov、.mkv、.ts 或 .3gp。
  2. 按「開啟」。上方資訊列會顯示寬度、高度、FPS、影格數與影片長度。
  3. 拖曳時間軸預覽；也可用「上一影格」「下一影格」，或輸入影格索引後按「跳到」。
  4. 預覽影格的時間是用 frame_index / FPS 計算，輸入框接受秒數、MM:SS 或 HH:MM:SS.mmm。

4. 單段切割：輸出一個片段

  1. 在「切割範圍」填寫開始與結束時間，例如 00:12:30.000 到 00:13:10.000。
  2. 或在預覽到目標影格後，按「用目前影格設定開始」與「用目前影格設定結束」。
  3. 選擇「單段：輸出一個起訖範圍」。
  4. 設定輸出資料夾與檔名前綴，例如 camera_01_event。
  5. 按「開始切割」。輸出會類似：
video_splitter/data/output/camera_01_event_0001.mp4

5. 批次切割：長影片切成多段

  1. 設定整體開始與結束時間；預設是整支影片。
  2. 選擇「批次：依固定長度切成多段」。
  3. 填寫「每段秒數」，例如 60。
  4. 填寫「段與段起點間隔秒數」。設為 60 代表不重疊；設為 30 代表每 30 秒開始一段、每段 60 秒，會有 30 秒重疊。
  5. 按「開始切割」。最後不足完整長度的尾段仍會輸出，只要至少有一個影格。

例如 0 到 125 秒、每段 60 秒、間隔 60 秒，會產生 0–60、60–120、120–125 三段。

6. 檔案管理與取消

  • 未勾選「允許覆寫同名檔案」時，工具會自動加上 _001、_002，避免蓋掉已有片段。
  • 勾選覆寫前，請確認輸出資料夾就是此次工作的目標資料夾。
  • 切割在背景執行，進度列會更新；按「取消」會在目前影格寫入後停止，已完成的片段會保留。
  • 輸出影片是影像重新編碼結果，不保留原始音訊。若影片有不同的旋轉 metadata，也應以輸出畫面實際方向為準。

7. 接到 YOLO11 資料製作流程

切割完成後，把片段交給上一個子專案：

python dataset_builder/scripts/extract_frames.py `
  --video video_splitter/data/output/camera_01_event_0001.mp4 `
  --output dataset_builder/data/staging `
  --every-seconds 1

python dataset_builder/gui/annotator_gui.py `
  --video video_splitter/data/output/camera_01_event_0001.mp4 `
  --output data/processed/video_yolo `
  --split train

同一場次的切割片段不要分散到 train、val、test。為避免資料洩漏，應先按日期、攝影機或場次決定 split，再把同一來源場次的片段放在同一個 split。

常見問題

現象處理方式
影片無法開啟，尤其是 .3gp確認目前 Prompt 是正確的 Anaconda 環境，先用 import cv2 測試；再確認檔案可由一般播放器播放。.3gp 是否能解碼取決於 OpenCV/FFmpeg 後端是否包含 3GPP codec。
輸出資料夾沒有檔案確認結束時間大於開始時間、輸出資料夾可寫入，且影片編碼器可建立 mp4v。
輸出影片沒有聲音這是本工具的預期行為；影像資料製作不受影響。需要音訊時請用另外的 FFmpeg 影音流程。
切割速度較慢此工具會重新編碼，速度取決於 CPU、影片解析度與片段數；批次重疊越多，重讀影格的工作量越大。
GitHub repository 變大不要提交原始影片或輸出影片；video_splitter/.gitignore 已排除常見影片副檔名與 data/output。

Git 提交流程

git status
git add video_splitter README.md
git commit -m "Add Tkinter video splitter subproject"
git push

提交前可執行 python -m compileall video_splitter/src video_splitter/gui video_splitter/tests，並在已安裝完整依賴的 Windows 環境執行 pytest。
